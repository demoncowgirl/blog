What is a framework?

A framework is built on top of a programming language to aid in a certain type of computer program, such as a web server or a mobile application, but a framework is not necessary for building applications and there can be more than one framework for each language. There are also frameworks for software applications and frameworks for web development. For Ruby, the web framework is Rails. Django and Flask are web frameworks built on top of Python., and React, Angular and Vue are frameworks built on top of Javascript. Frameworks help to increase performance and to extend capabilities as well as provide libraries. Some also offer APIs and modals.

What is a library?

A library contains pieces of shared code that is usually, but not always, written in the same code as the framework. It can be reused so that certain functionality does not need to be built from scratch. I think of them as shortcuts. Libraries are also referred to as packages or modules, adding to a bit of my confusion. They can also be a part of the coding language(standard library), or come from an external source (external library).

So there is only one framework used in an application, but there can be many libraries.. Changing the programming language and framework is very hard once the coding application has been started, although it is easier to change the framework, as some pieces of code may be swapped to the new framework.

Besides libraries, web application frameworks can include APIs, scaffolding, AJAX, caching, security and compilers. JavaScript frameworks include Angular, jQuery, Node.JS, ExpressJS and React JS. Express is a back-end library—I’ve also seen it referred to as a framework which contributes to my confusion -- that runs on the node.js platform and Bootstrap is a front-end framework. Vue and Express are JavaScript libraries. CSS frameworks include the pre-compilers LESS and Sass.

What is an API?

An API, (Application Programming Interface) like openweathermap.org, interprets data provides developers with this information in a way that can be used. It is the middle man between you and a whole lot of data! You send a request for information. It is then retrieved, interpreted and returned to you in a way that can be understood. Facebook is an API. An API also provides some security, as the data shared is in small packets and is only what you need.
